## 一、数据库设计

### 新增数据表

#### 1. PresalesActivity (售前活动表)
```
model PresalesActivity {
  id               String          @id @default(uuid())
  title            String          // 活动标题
  type             String          // demo/poc/training/seminar
  description      String?         @db.Text
  customerId       String?         // 关联客户(可选)
  location         String?
  startTime        DateTime
  endTime          DateTime
  status           ActivityStatus  @default(draft) // draft/pending/approved/ongoing/completed/cancelled
  approvalStatus   ApprovalStatus  @default(none) // none/pending/approved/rejected
  approvalNotes    String?         @db.Text
  approvedById     String?
  approvedAt       DateTime?
  createdById      String
  createdAt        DateTime        @default(now())
  updatedAt        DateTime        @updatedAt
  
  // Relations
  customer         Customer?       @relation(...)
  createdBy        User            @relation(...)
  approvedBy       User?           @relation(...)
  qrCodes          ActivityQrCode[]
  signIns          ActivitySignIn[]
  questions        ActivityQuestion[]
}
```

#### 2. ActivityQrCode (活动二维码表)
```
model ActivityQrCode {
  id           String   @id @default(uuid())
  activityId   String
  codeType     String   // exclusive/generic
  qrCodeUrl    String   // 二维码图片URL
  qrCodeData   String   // 二维码数据
  validFrom    DateTime
  validUntil   DateTime?
  isActive     Boolean  @default(true)
  scanCount    Int      @default(0)
  createdAt    DateTime @default(now())
  
  activity     PresalesActivity @relation(...)
  signIns      ActivitySignIn[]
}
```

#### 3. ActivitySignIn (活动签到表)
```
model ActivitySignIn {
  id             String   @id @default(uuid())
  activityId     String
  qrCodeId       String
  customerId     String?  // 关联已有客户
  customerName   String
  phone          String
  email          String?
  company        String?
  title          String?  // 职位
  isNewCustomer  Boolean  @default(true)
  notes          String?  @db.Text
  signedAt       DateTime @default(now())
  createdById    String?
  createdAt      DateTime @default(now())
  
  activity       PresalesActivity @relation(...)
  qrCode         ActivityQrCode @relation(...)
  customer       Customer? @relation(...)
  questions      ActivityQuestion[]
}
```

#### 4. ActivityQuestion (活动问题表)
```
model ActivityQuestion {
  id           String   @id @default(uuid())
  activityId   String
  signInId     String
  customerId   String?  // 关联客户
  question     String   @db.Text
  category     String?  // product/pricing/technical/others
  priority     String?  // high/medium/low
  aiAnalysis   Json?    // AI分析结果
  status       String   @default("pending") // pending/answered/closed
  answer       String?  @db.Text
  answeredAt   DateTime?
  answeredBy   String?
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt
  
  activity     PresalesActivity @relation(...)
  signIn       ActivitySignIn @relation(...)
  customer     Customer? @relation(...)
}
```

### API端点设计

#### 活动管理
- `POST /api/v1/presales/activities` - 创建活动
- `GET /api/v1/presales/activities` - 获取活动列表
- `GET /api/v1/presales/activities/:id` - 获取活动详情
- `PUT /api/v1/presales/activities/:id` - 更新活动
- `DELETE /api/v1/presales/activities/:id` - 删除活动
- `PATCH /api/v1/presales/activities/:id/status` - 更新活动状态

#### 审批流程
- `POST /api/v1/presales/activities/:id/submit-approval` - 提交审批
- `POST /api/v1/presales/activities/:id/approve` - 审批通过
- `POST /api/v1/presales/activities/:id/reject` - 审批拒绝

#### 二维码管理
- `POST /api/v1/presales/activities/:id/qrcodes` - 生成二维码
- `GET /api/v1/presales/activities/:id/qrcodes` - 获取活动二维码列表
- `GET /api/v1/presales/qrcodes/:qrCodeId` - 获取二维码详情

#### 签到功能
- `POST /api/v1/presales/sign-in` - 扫码签到(无需认证)
- `GET /api/v1/presales/activities/:id/sign-ins` - 获取签到记录
- `GET /api/v1/presales/sign-ins/:id` - 获取签到详情

#### 问题管理
- `POST /api/v1/presales/sign-ins/:signInId/questions` - 提交问题
- `GET /api/v1/presales/activities/:id/questions` - 获取活动问题列表
- `PATCH /api/v1/presales/questions/:id` - 更新问题状态
- `POST /api/v1/presales/questions/:id/answer` - 回答问题
- `POST /api/v1/presales/questions/classify` - AI分类问题

### 1. 二维码生成流程

```typescript
// 使用qrcode库生成二维码
import QRCode from 'qrcode';

// 生成二维码数据
const qrData = JSON.stringify({
  activityId,
  qrCodeId,
  type: 'presales_signin',
  timestamp: Date.now()
});

// 生成二维码图片URL
const qrCodeUrl = await QRCode.toDataURL(qrData);
```

### 2. 签到客户匹配逻辑
```typescript
async handleSignIn(data: SignInInput) {
  // 1. 检查客户是否存在(按手机号或邮箱)
  let customer = await findCustomerByPhoneOrEmail(data.phone, data.email);
  
  // 2. 如果不存在,创建新客户
  if (!customer) {
    customer = await createCustomer({
      name: data.customerName,
      phone: data.phone,
      email: data.email,
      company: data.company,
      source: 'presales_activity'
    });
  } else {
    // 3. 如果存在,更新客户信息
    await updateCustomer(customer.id, {
      lastContactDate: new Date()
    });
  }
  
  // 4. 创建签到记录
  const signIn = await createSignInRecord({
    activityId: data.activityId,
    customerId: customer.id,
    ...data
  });
  
  return signIn;
}
```

### 3. AI问题分类逻辑
```typescript
async classifyQuestion(question: string): Promise<QuestionClassification> {
  // 调用AI服务进行分类
  const result = await aiService.classify({
    text: question,
    categories: ['product', 'pricing', 'technical', 'implementation', 'others'],
    context: 'presales_activity'
  });
  
  return {
    category: result.category,
    priority: result.priority,
    keywords: result.keywords,
    suggestedAnswer: result.suggestedAnswer
  };
}
```

## 六、技术要点

### 依赖库
- 后端: `qrcode` (二维码生成), `@prisma/client`
- 前端: `react-qr-reader` (可选,扫码功能)

### 安全考虑
1. 签到接口无需认证,但需验证二维码有效性
2. 二维码设置有效期防止滥用
3. 敏感操作需要认证和权限验证

### 性能优化
1. 二维码图片使用CDN存储
2. 签到记录分页查询
3. 问题分类使用异步处理
