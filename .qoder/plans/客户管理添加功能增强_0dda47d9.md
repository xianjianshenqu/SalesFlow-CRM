# 客户管理"添加客户"功能增强实现计划

## 一、数据库层修改

### 1.1 更新Prisma Schema (crm-backend/prisma/schema.prisma)

在Customer模型中添加以下字段：
```
model Customer {
  // ... 现有字段 ...
  
  // 新增客户分类字段
  customerType     String   @default("non_user") // user, non_user, valid_non_user, invalid_non_user
  
  // 新增企业信息字段
  companyFullName  String?  // 企业全称
  creditCode      String?  // 统一社会信用代码
  registeredCapital Decimal? @db.Decimal(15, 2) // 注册资本
  establishDate   DateTime? // 成立日期
  businessScope   String?  @db.Text // 经营范围
  legalPerson     String?  // 法人代表
  companyStatus   String?  // 企业状态（存续、注销等）
}
```

### 1.2 数据库迁移
```bash
npx prisma migrate dev --name add_customer_type_and_company_info
```

## 四、字段设计详细说明

### 4.1 基础信息字段（必填标记*）
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 企业名称 | String | 是 | 可搜索选择或手动输入 |
| 企业简称 | String | 是 | 最大10个字符 |
| 联系人 | String | 是 | 主联系人姓名 |
| 联系电话 | String | 否 | 手机或座机 |
| 邮箱 | String | 否 | 企业邮箱 |
| 地址 | String | 否 | 详细地址 |
| 城市 | String | 否 | 所在城市 |

### 4.2 企业信息字段（从企业查询自动填充或手动输入）
| 字段 | 类型 | 说明 |
|------|------|------|
| 统一社会信用代码 | String | 唯一标识 |
| 注册资本 | Decimal | 单位：万元 |
| 成立日期 | DateTime | |
| 法人代表 | String | |
| 经营范围 | Text | |
| 企业状态 | String | 存续/注销/吊销等 |

### 4.3 客户分类定义
| 类型 | 值 | 说明 | 建议操作 |
|------|------|------|---------|
| 用户 | user | 已购买产品/服务 | 重点维护，挖掘增购 |
| 非用户 | non_user | 未购买但有接触 | 持续跟进 |
| 有效非用户 | valid_non_user | 有购买潜力 | 重点跟进 |
| 无效非用户 | invalid_non_user | 无购买意向 | 低优先级或归档 |

## 六、技术要点

1. **企业搜索防抖**：输入延迟300ms后触发搜索
2. **表单分步引导**：通过Tab切换展示不同信息区域
3. **智能填充**：选择企业后自动填充相关字段，支持手动修改
4. **验证机制**：
   - 企业简称：必填，最大10字符
   - 联系人：必填
   - 邮箱：格式验证
   - 手机号：11位数字验证

5. **UI交互优化**：
   - 搜索时显示loading状态
   - 无匹配结果时显示友好提示
   - 表单提交时禁用按钮防止重复提交
