### 任务4: 更新API服务层
**目标**: 确保前端API调用正确

**涉及文件**:
- `crm-frontend/src/services/api.ts` - 添加报告相关API（如尚未存在）

**具体变更**:
1. 在api.ts中添加AI报告相关API:
   ```typescript
   export const aiApi = {
     generateReport: (type: 'daily' | 'weekly', date?: string) =>
       api.post('/ai/reports/generate', { type, date }),
     getReports: (params?: { type?: string; page?: number; limit?: number }) => {
       const query = new URLSearchParams();
       if (params?.type) query.set('type', params.type);
       if (params?.page) query.set('page', String(params.page));
       if (params?.limit) query.set('limit', String(params.limit));
       return api.get(`/ai/reports?${query}`);
     },
     getReportById: (id: string) => api.get(`/ai/reports/${id}`),
   };
   ```

### 任务5: 导航菜单顺序优化
**目标**: 按照销售人员工作流程优化菜单顺序

**建议的新菜单顺序**:
1. 企业知识库 (/knowledge) - 产品资料、合同模板、定价表等销售资料库
2. 工作台 (/) - 每日工作概览、待办事项
3. 智能日程 (/schedule) - 日程管理 + AI日报周报
4. 陌生拜访AI助手 (/cold-visit) - 新客户开拓、企业分析
5. 客户管理 (/customers) - 客户跟进、商机管理
6. 客户地图 (/map) - 拜访规划、客户分布
7. 销售漏斗 (/funnel) - 商机阶段管理
8. 售前中心 (/presales) - 售前活动、客户签到
9. 商务方案 (/proposals) - 方案设计、审批流程
10. AI录音分析 (/ai-audio) - 通话录音智能分析
11. 售后服务 (/service) - 客户支持、工单管理
12. 回款统计 (/payments) - 财务管理、收款跟踪
13. 团队协作 (/team) - 团队成员、业绩统计

**涉及文件**:
- `crm-frontend/src/components/layout/Sidebar.tsx`

1. **类型导入规范**: 所有类型导入必须使用`import type`语法
2. **API URL**: 使用`import.meta.env.VITE_API_BASE_URL`，禁止硬编码
3. **后端服务**: 修改前确保后端服务已启动
4. **功能完整性**: 确保原有功能在迁移后保持完整
5. **用户体验**: 保持界面风格一致性
