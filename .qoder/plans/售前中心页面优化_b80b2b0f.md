# 售前中心页面优化计划

## 问题分析

基于页面截图和代码分析，当前存在以下问题：

1. **统计卡片区域**："审批中活动"卡片出现"ACTION"文字溢出边界
2. **快速操作区域**："创建活动"按钮图标与其他三个功能图标尺寸不一致
3. **今日活动卡片**：占用空间过大，需要更加紧凑
4. **售前资源区域**：仅显示头像和状态，缺少工作内容和联系功能

## 优化任务清单

### 任务1：统计卡片区域优化
**目标文件**：`crm-frontend/src/pages/PreSales/index.tsx`

**修改内容**：
- 修复"审批中活动"卡片的文字溢出问题
- 将第4个卡片的图标从 `pending_action` 改为更合适的图标（如 `schedule` 或 `event_note`）
- 调整卡片内边距和文字大小，确保内容不会溢出

**关键代码位置**：
- `BusinessStats` 组件中第100-108行的第4个统计卡片配置
- 修改 `icon` 属性和 `label` 属性

### 任务2：快速操作区域图标一致性调整
**目标文件**：`crm-frontend/src/pages/PreSales/index.tsx`

**修改内容**：
- 统一四个快速操作按钮的图标尺寸
- 当前"创建活动"按钮图标（`event_add`）尺寸与其他三个不一致
- 调整图标容器大小和图标字体大小

**关键代码位置**：
- `QuickActions` 组件（第420-451行）
- 调整图标容器的 `w-11 h-11` 和图标的 `text-lg` 样式

### 任务3：今日活动卡片尺寸优化
**目标文件**：`crm-frontend/src/pages/PreSales/index.tsx`

**修改内容**：
- 缩小今日活动卡片的整体尺寸
- 减少内边距（`p-5` 调整为 `p-4` 或 `p-3`）
- 缩小标题字体大小（`text-lg` 调整为 `text-base`）
- 紧凑化信息展示区域

**关键代码位置**：
- `TodayActivities` 组件（第143-219行）
- 调整各个元素的 padding 和 margin

### 任务4：售前资源管理功能增强
**目标文件**：`crm-frontend/src/pages/PreSales/index.tsx`

**修改内容**：
- 扩展 `mockResources` 数据结构，添加工作内容字段
- 重新设计资源卡片布局：
  - 左侧：头像 + 姓名 + 职位
  - 中间：当前工作内容/任务描述
  - 右侧：状态指示器 + 联系按钮
- 添加联系功能按钮（电话、消息、邮件）
- 使用更丰富的卡片设计替代当前的简单标签

**关键代码位置**：
- `ResourceOverview` 组件（第370-416行）
- `mockResources` 数据定义（第42-48行）

### 任务5：整体视觉美化
**目标文件**：`crm-frontend/src/pages/PreSales/index.tsx`

**修改内容**：
- 统一间距系统（使用一致的 gap 和 padding）
- 优化颜色搭配，确保视觉层次清晰
- 调整阴影和圆角，提升卡片质感
- 优化响应式布局
